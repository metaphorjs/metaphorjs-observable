MetaphorJs.Observable

A javascript event system

v1.2

class Observable

  • @author Ivan Kuindzhi

A javascript event system implementing multiple patterns: observable, collector and pipe.

Observable:

var o = new Observable;
o.on("event", function(x, y, z){ console.log([x, y, z]) });
o.trigger("event", 1, 2, 3); // [1, 2, 3]

Collector:

var o = new Observable;
o.createEvent("collectStuff", "all");
o.on("collectStuff", function(){ return 1; });
o.on("collectStuff", function(){ return 2; });
var results = o.trigger("collectStuff"); // [1, 2]

Pipe:

var o = new Observable;
o.createEvent("some-job", "pipe");
o.on("some-job", function(value){
    return value + value;
});
o.on("some-job", function(value){
    return value * value;
});

var result = o.trigger("some-job", 1); // 4

Methods

  • createEvent (name, options)  -> ObservableEvent

    • required
      string name

      Event name

    • object | string | boolean options

      Options object or returnResult value. All options are optional.

      • string | boolean returnResult

        false -- return first 'false' result and stop calling listeners after that
        true -- return first 'true' result and stop calling listeners after that
        "all" -- return all results as array
        "concat" -- merge all results into one array (each result must be array)
        "merge" -- merge all results into one object (each result much be object)
        "pipe" -- pass return value of previous listener to the next listener. Only first trigger parameter is being replaced with return value, others stay as is.
        "first" -- return result of the first handler (next listener will not be called)
        "nonempty" -- return first nonempty result
        "last" -- return result of the last handler (all listeners will be called)

      • boolean autoTrigger

        once triggered, all future subscribers will be automatically called with last trigger params

        var o = new Observable;
o.createEvent("auto", {autoTrigger: true});
// trigger first
o.trigger("auto", 1, 2);
// subscribe later
o.on("auto", function(a, b){
    console.log(a, b); // immediately logs 1, 2
});
      • function triggerFilter (listener, arguments)  -> boolean

        This function will be called each time event is triggered. Return false to skip listener.

        var o = new Observable;
o.createEvent("filtered", {triggerFilter: function(l, args){
    if (l.always) {
        return true;
    }
    if (l.param == args[0]) {
        return true;
    }
    return false;
}});

o.on("filtered", function(){
    console.log("always")
}, null, {
    always: true
});

o.on("filtered", function(){
    console.log("param");
}, null, {
    param: 1
});


o.trigger("filtered", 2); // "always"
o.trigger("filtered", 1); // "always", "param"
        • object listener

          This object contains all information about the listener, including all data you provided in options while subscribing to the event.

        • array arguments
      • object filterContext

        triggerFilter's context

      • boolean expectPromises

        Expect listeners to return Promises. If returnResult is set, promises will be treated as return values unless resolvePromises is set.

      • boolean resolvePromises

        In pair with expectPromises and returnResult this option makes trigger function wait for promises to resolve. All or just one depends on returnResult mode. "pipe" mode makes promises resolve consequentially passing resolved value to the next promise.

  • on (name, fn, context, options)

    • @access public

    Subscribe to an event or register collector function.

    • required
      string name

      Event name. Use '*' to subscribe to all events.

    • required
      function fn ()

      Callback function

    • object context

      "this" object for the callback function

    • object options

      You can pass any key-value pairs in this object. All of them will be passed to triggerFilter (if you're using one).

      • boolean first
        • @default false

        True to prepend to the list of handlers

      • number limit
        • @default 0

        Call handler this number of times; 0 for unlimited

      • number start
        • @default 1

        Start calling handler after this number of calls. Starts from 1

      • array append

        Append parameters

      • array prepend

        Prepend parameters

      • boolean allowDupes

        allow the same handler twice

      • boolean | int async

        run event asynchronously. If event was created with expectPromises: true, this option is ignored.

mixin Observable

Mixin adds observable features to the host object. It adds 'callback' option to the host config. See $beforeInit. Mixin is designed for MetaphorJs class system.

var MyClass = defineClass({
    $class: "MyClass",
    $mixins: ["mixin.Observable"],
    $init: function() {
        this.trigger("someEvent", 123);
    },

    doSomeAction: function() {
        // does some action
        this.trigger("anotherEvent", 234);
    }
});

var ParentClass = defineClass({
    $init: function() {
        this.my = new MyClass({
            callback: {
                context: this,
                someEvent: this.onSomeEvent
            }
        });
        this.my.on("anotherEvent", this.onAnotherEvent, this);
        this.my.doSomeAction();
    },

    onSomeEvent: function(value) {
        // value == 123
    },

    onAnotherEvent: function(value) {
        // value == 234
    }
});

var p = new ParentClass;
